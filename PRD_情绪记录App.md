## 一、产品概述

### 1.1 产品定位

MoodLog（心迹）是一款面向大众用户的情绪记录与自我觉察 iOS 应用。通过极简的情绪打卡、丰富的活动标签体系、数据可视化分析，帮助用户建立情绪觉察习惯，理解情绪波动规律，提升心理健康水平。

### 1.2 产品愿景

让每一个人都能轻松记录情绪、理解自我，成为全球最受欢迎的情绪健康陪伴工具。

### 1.3 产品目标

| 指标 | 目标值 | 时间窗口 |
|------|--------|----------|
| DAU | 1,000万+ | 上线12个月 |
| 次日留存率 | ≥45% | 上线3个月 |
| 7日留存率 | ≥25% | 上线3个月 |
| 情绪记录频次 | ≥3次/日/活跃用户 | 上线6个月 |
| 付费转化率 | ≥5% | 上线6个月 |
| App Store 评分 | ≥4.7 | 持续 |

## 二、用户分析

### 2.1 目标用户画像

| 画像 | 年龄段 | 特征描述 | 核心需求 |
|------|--------|----------|----------|
| 职场压力族 | 25-35岁 | 一线城市白领，工作压力大，情绪波动频繁 | 快速记录情绪，释放压力 |
| 情感困扰者 | 18-35岁 | 经历恋爱、婚姻中的情感波折 | 标记情感事件，追踪情绪变化 |
| 自我成长者 | 20-40岁 | 关注心理健康，有自我觉察意识 | 情绪数据分析，成长轨迹 |
| 学生群体 | 16-24岁 | 学业压力、社交焦虑 | 简单易用，社交化表达 |
| 心理咨询辅助 | 25-50岁 | 正在接受或已完成心理咨询 | 为咨询师提供情绪数据支持 |

### 2.2 用户场景

1. **晨起打卡**：起床后花10秒记录当前情绪状态，开启新的一天
2. **情绪波动时**：工作中突然焦虑/愤怒，打开App快速记录并选择触发标签
3. **情感事件后**：经历争吵、分手、想离婚等事件后，记录情绪和事件标签
4. **睡前复盘**：回顾一天情绪变化，写一段简短反思
5. **周/月回顾**：查看情绪趋势图，发现情绪规律

| 属性 | 说明 |
|------|------|

### 4.3 数据洞察（P0基础版 / P1高级版）

#### 4.3.1 情绪趋势图（免费）

| 图表类型 | 说明 |
|----------|------|
| 周趋势 | 最近7天情绪均值折线图 |
| 月趋势 | 最近30天情绪均值折线图 |
| 情绪分布 | 饼图展示各情绪类型占比 |
| 标签频次 | Top10 活动标签柱状图 |

#### 4.3.2 活动标签关联分析（付费）

- 标签与情绪的关联热力图
- "想离婚"标签下的情绪分布
- "想分手"标签下的情绪强度变化趋势
- 特定标签出现频率的时间趋势
- 标签共现分析（如"吵架了"常与"冷战"共现）

#### 4.3.3 AI 情绪洞察（付费）

- 基于用户情绪数据生成周/月洞察报告
- 识别情绪触发模式："您在'工作日'的情绪普遍低于'周末'"
- 提供个性化建议："当您标记'焦虑'时，'散步'标签后的情绪平均提升2.3分"
- 情绪预警：连续3天负面情绪强度>7时，推送关怀提醒

### 4.4 发现/内容模块（P1）

#### 4.4.1 情绪健康知识库

- 基于用户情绪标签推荐相关文章
- "想离婚"→推荐婚姻沟通、法律常识、心理调适文章
- "想分手"→推荐情感决策、自我关爱、社交重建文章
- 文章由专业心理咨询师审核
- 支持收藏与分享

#### 4.4.2 冥想与呼吸练习

- 内置5组呼吸练习（4-7-8呼吸法、盒式呼吸等）
- 内置3段引导冥想音频（5min/10min/15min）
- 练习完成后自动记录一条"自我关怀"标签的情绪

#### 4.4.3 社区匿名分享

- 用户可选择将情绪记录匿名分享到社区
- 社区按标签分类浏览
- 支持点赞、拥抱（hug）、评论
- 严格内容审核机制（AI+人工）
- 禁止自残/自杀相关内容的详细描述，触发危机干预

#### 4.4.4 危机干预入口

- 当检测到高风险情绪模式时，弹出关怀弹窗
- 提供全国心理援助热线：400-161-9995
- 提供紧急联系设置（用户预设的紧急联系人）
- 一键拨打功能

## 五、非功能需求

### 5.1 性能要求

| 指标 | 要求 |
|------|------|
| App启动时间 | 冷启动 ≤ 2s，热启动 ≤ 0.5s |
| 打卡完成时间 | 从打开App到完成打卡 ≤ 3s |
| 页面加载时间 | ≤ 1s |
| 内存占用 | ≤ 150MB |
| 包体积 | ≤ 80MB |
| 崩溃率 | ≤ 0.1% |
| ANR率 | ≤ 0.3% |

### 5.2 兼容性

- iOS 15.0 及以上
- iPhone 8 及以上机型
- 支持深色模式
- 支持动态字体（Dynamic Type）
- 支持VoiceOver无障碍

### 5.3 数据安全

- 所有用户数据本地加密存储
- 网络传输使用TLS 1.3
- 不收集用户原始情绪数据用于广告
- 符合GDPR、个人信息保护法要求
- 通过Apple App Store审核的数据隐私要求

### 5.4 可用性

- 离线模式：无网络时完整支持打卡、日历查看
- 数据同步：网络恢复后自动同步至iCloud
- 容灾：本地数据损坏时支持iCloud恢复

## 六、UI/UX 设计规范

### 6.1 设计原则

1. **极简优先**：打卡路径最短化，3秒完成记录
2. **温暖治愈**：圆角、柔和色彩、微动效，营造安全感
3. **隐私尊重**：不打扰、不评判、不泄露
4. **渐进展示**：基础功能零门槛，高级功能按需发现

### 6.2 视觉规范

| 元素 | 规范 |
|------|------|
| 主色调 | #6C5CE7（静谧紫） |
| 辅助色 | #00B894（治愈绿） |
| 背景色 | 亮色#FAFAFA / 暗色#1A1A2E |
| 圆角 | 16pt（卡片）/ 12pt（按钮）/ 20pt（弹窗） |
| 字体 | 系统SF Pro / 中文PingFang SC |
| 图标风格 | 填充式emoji + 线性图标混合 |
| 动效 | 弹性缓动，时长200-400ms |

### 6.3 关键页面交互流程

**情绪打卡主流程**：
```
打开App → 首页（情绪轮盘）
  → 选择一级情绪 → 展开二级情绪+强度滑块
  → （可选）选择活动标签
  → （可选）添加备注
  → 点击"记录"→ 成功动画+打卡完成
  → 返回首页/查看日历
```

**快捷打卡流程**：
```
Widget/Siri → 选择情绪 → 自动使用上次标签+强度 → 打卡完成
```

## 七、技术方案概要

### 7.1 技术栈

| 层级 | 技术选型 |
|------|----------|
| 开发语言 | Swift 5.9+ |

### 7.2 核心模块

| 模块 | 职责 |
|------|------|
| MoodKit | 情绪模型、标签模型、打卡逻辑 |
| CalendarKit | 日历渲染、时间线视图 |
| InsightKit | 数据分析、图表渲染、AI洞察 |
| SyncKit | iCloud同步、冲突解决 |
| PushKit | 推送管理、智能提醒 |
| SecurityKit | 加密存储、生物识别解锁 |
| WidgetKit | 小组件渲染与数据更新 |

### 7.3 数据模型

```
MoodRecord {
    id: UUID
    moodType: MoodType          // 一级情绪
    moodSubType: MoodSubType    // 二级情绪
    intensity: Int              // 1-10
    tags: [ActivityTag]         // 活动标签
    note: String?               // 备注
    images: [ImageData]?        // 图片
    createdAt: Date             // 记录时间
    updatedAt: Date             // 更新时间
    isSynced: Bool              // 是否已同步
}

ActivityTag {
    id: UUID
    name: String                // 标签名
    category: TagCategory       // 分类
    emoji: String               // emoji图标
    isCustom: Bool              // 是否自定义
    usageCount: Int             // 使用次数
    createdAt: Date
}
```

## 十一、风险与应对
